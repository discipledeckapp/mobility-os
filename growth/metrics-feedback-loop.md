# Metrics and Feedback Loop

## 1. Core Metrics

Track weekly:

- leads sourced
- leads enriched
- outreach sent
- reply rate
- positive reply rate
- demos booked
- signups
- first driver added
- first driver verified
- first assignment created

## 2. Funnel View

### Top of funnel

- raw leads sourced
- qualified leads
- A leads

### Mid funnel

- initial messages sent
- replies
- positive replies
- demos booked

### Activation

- signup
- first driver added
- first driver verified
- first assignment created

## 3. Learning Questions

Review weekly:

- which ICP replied most?
- which pain angle produced the most replies?
- which CTA produced the most positive replies?
- which content topic generated the strongest conversations?
- where are signups failing to activate?

## 4. Message Learning System

For every positive reply:

- tag the message angle
- tag the ICP
- tag the specific phrase that likely resonated

For every no-reply cluster:

- tag the angle
- tag the channel
- tag the ICP

After two weeks:

- keep top 3 angles
- rewrite or kill weak angles

## 5. Content Learning System

If a content topic creates:

- strong clicks
- inbound replies
- repeat objections

Then:

- build more content around that pain
- use the same language in outreach
- create a landing page for that angle

## 6. Activation Learning System

If signups stall before:

- first driver added
- first driver verified
- first assignment created

Then:

- ask what step felt unclear
- capture the exact language users used
- feed those phrases back into:
  - homepage copy
  - landing pages
  - demo narrative
  - outreach promises

## 7. Weekly Review Prompt

```text
Review Mobiris growth and activation performance.

Data provided:
- weekly lead numbers
- outreach numbers
- replies
- demos
- signups
- first driver added
- first driver verified
- first assignment created
- notes from calls or support

Return:
1. bottleneck in the funnel
2. best ICP right now
3. best message angle right now
4. activation risk right now
5. 3 changes to make next week
```
